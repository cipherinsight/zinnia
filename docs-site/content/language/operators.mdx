# Operators and Built-in Functions

All operations in Zinnia ultimately compile to field arithmetic over a prime field F_p.
This page documents every operator and function available in the language.

## Arithmetic Operators

| Operator | Description         | Integer | Float |
|----------|---------------------|---------|-------|
| `+`      | Addition            | Yes     | Yes   |
| `-`      | Subtraction         | Yes     | Yes   |
| `*`      | Multiplication      | Yes     | Yes   |
| `/`      | Division            | Yes     | Yes   |
| `//`     | Floor division      | Yes     | Yes   |
| `%`      | Modulo              | Yes     | Yes   |
| `**`     | Exponentiation      | Yes     | Yes   |
| `@`      | Matrix multiply     | NDArray | NDArray |

```python
a = 10 + 3      # 13
b = 10 - 3      # 7
c = 10 * 3      # 30
d = 10 / 3      # 3 (integer division) or 3.333... (float)
e = 10 // 3     # 3
f = 10 % 3      # 1
g = 2 ** 10     # 1024

# Matrix multiplication
A = NDArray.asarray([[1, 2], [3, 4]])
B = NDArray.asarray([[5, 6], [7, 8]])
C = A @ B       # [[19, 22], [43, 50]]
```

## Comparison Operators

| Operator | Description              |
|----------|--------------------------|
| `==`     | Equal                    |
| `!=`     | Not equal                |
| `<`      | Less than                |
| `<=`     | Less than or equal       |
| `>`      | Greater than             |
| `>=`     | Greater than or equal    |

Comparison operators return `Bool` values. They work on scalars and are broadcast
element-wise on arrays.

```python
is_positive = x > 0
is_equal = a == b
in_range = x >= lo and x <= hi
```

## Logical Operators

| Operator | Description       |
|----------|-------------------|
| `and`    | Logical AND       |
| `or`     | Logical OR        |
| `not`    | Logical NOT       |

```python
valid = x > 0 and y > 0
either = a or b
negated = not flag
```

In the circuit, `and` compiles to multiplication of boolean field elements, `or` to
`a + b - a * b`, and `not` to `1 - a`.

## Unary Operators

| Operator | Description       |
|----------|-------------------|
| `+x`    | Unary positive     |
| `-x`    | Unary negation     |

```python
pos = +x      # identity
neg = -x      # negation (field subtraction from 0)
```

## Built-in Functions

### General

| Function  | Description                                    |
|-----------|------------------------------------------------|
| `len(x)`  | Length of a list, tuple, or array              |
| `range(...)` | Integer range for iteration                |
| `sum(x)`  | Sum of elements                                |
| `any(x)`  | True if any element is truthy                  |
| `all(x)`  | True if all elements are truthy                |
| `min(x)`  | Minimum value                                  |
| `max(x)`  | Maximum value                                  |
| `abs(x)`  | Absolute value                                 |
| `pow(x, y)` | Exponentiation (equivalent to `x ** y`)     |
| `print(...)` | Debug output (no-op in circuit)             |

```python
n = len([1, 2, 3])          # 3

has_neg = any(v < 0 for v in data)
all_pos = all(v > 0 for v in data)

hi = max(3, 7)               # 7
a = abs(-5)                  # 5
p = pow(2, 10)               # 1024
```

### Cryptographic Functions

## Math Module Functions

The `math` module provides standard mathematical functions. These compile to
circuit-friendly approximations or exact field computations where applicable.

| Function       | Description                     |
|----------------|---------------------------------|
| `math.sin(x)`  | Sine                           |
| `math.cos(x)`  | Cosine                         |
| `math.tan(x)`  | Tangent                        |
| `math.sinh(x)` | Hyperbolic sine                |
| `math.cosh(x)` | Hyperbolic cosine              |
| `math.tanh(x)` | Hyperbolic tangent             |
| `math.exp(x)`  | Exponential (e^x)             |
| `math.log(x)`  | Natural logarithm              |
| `math.sqrt(x)` | Square root                    |
| `math.fabs(x)` | Absolute value (float)         |
| `math.inv(x)`  | Multiplicative inverse (1/x)   |

```python
import math

y = math.sin(3.14159 / 2)    # approximately 1.0
z = math.exp(1.0)             # approximately 2.718
r = math.sqrt(2.0)            # approximately 1.414
inv = math.inv(5.0)           # 0.2
```

## NumPy-Like Functions (np namespace)

Zinnia provides a `np` namespace with NumPy-compatible functions that operate on arrays.

### Array Creation

| Function                | Description                              |
|-------------------------|------------------------------------------|
| `np.zeros(shape)`       | Array of zeros                          |
| `np.ones(shape)`        | Array of ones                           |
| `np.eye(n, m=None)`     | Identity/diagonal matrix                |
| `np.identity(n)`        | Square identity matrix                  |
| `np.array(data)`        | Create array from data                  |
| `np.arange(start, stop, step)` | Array of evenly spaced values   |
| `np.linspace(start, stop, num)` | Array of linearly spaced values |

```python
z = np.zeros((3, 3))

I = np.identity(5)

ls = np.linspace(0.0, 1.0, 11)
```

### Array Manipulation

| Function                        | Description                        |
|---------------------------------|------------------------------------|

## List Methods

| Method              | Description                                 |
|---------------------|---------------------------------------------|
| `list.append(x)`    | Add an element to the end                  |
| `list.extend(xs)`   | Extend with elements from an iterable      |
| `list.insert(i, x)` | Insert element at position i              |
| `list.pop(i=-1)`    | Remove and return element at position i    |
| `list.copy()`       | Return a shallow copy                      |
| `list.clear()`      | Remove all elements                        |
| `list.reverse()`    | Reverse the list in place                  |
| `list.index(x)`     | Return index of first occurrence of x      |
| `list.count(x)`     | Count occurrences of x                     |
| `list.remove(x)`    | Remove first occurrence of x               |

```python
xs = [1, 2, 3]
xs.append(4)          # [1, 2, 3, 4]
xs.extend([5, 6])     # [1, 2, 3, 4, 5, 6]
xs.insert(0, 0)       # [0, 1, 2, 3, 4, 5, 6]

n = xs.count(1)       # 1
xs.reverse()          # [5, 4, 3, 2, 1, 0]
```

## A Note on Field Arithmetic

All Zinnia operations compile to arithmetic over a prime field F_p, where p is determined
by the zk-SNARK backend. This has several implications:

- **Integer overflow**: integers are elements of F_p, so arithmetic wraps modulo p.
  In practice, p is very large (e.g., 2^254 for BN254), so overflow is rarely a concern
  for typical program values.
- **Division**: division computes the multiplicative inverse in F_p, not truncated
  integer division. The `//` operator provides floor division semantics via additional
  constraints.
- **Comparison**: ordering comparisons (`<`, `>`, `<=`, `>=`) require range checks,
  which add constraints to the circuit.
- **Float precision**: floating-point values are encoded as fixed-point or rational
  representations over F_p, with precision bounded by the encoding scheme.
