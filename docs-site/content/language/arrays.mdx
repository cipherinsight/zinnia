# Arrays

Zinnia provides two array types for N-dimensional numeric computation: `NDArray` for
fixed-size arrays and `DynamicNDArray` for arrays with runtime-determined shapes. Both
types offer a NumPy-compatible interface.

## NDArray -- Fixed-Size N-Dimensional Arrays

`NDArray` is the primary array type. Its shape is fully known at compile time, which
allows the compiler to generate an exactly sized circuit.

### Creation

#### NDArray.zeros

Create an array filled with zeros.

```python
vec = NDArray.zeros((5,))                       # 1-D, 5 elements
mat = NDArray.zeros((3, 4))                     # 3x4 matrix
mat_f = NDArray.zeros((3, 4), dtype=Float)      # 3x4 float matrix
```

#### NDArray.ones

Create an array filled with ones.

```python
vec = NDArray.ones((5,))
mat = NDArray.ones((2, 3), dtype=Float)
```

#### NDArray.identity

Create a square identity matrix.

```python
I = NDArray.identity(4)    # 4x4 identity matrix
```

#### NDArray.eye

Create a matrix with ones on the diagonal and zeros elsewhere. Supports non-square
matrices.

```python
I = NDArray.eye(3, 3)     # 3x3 identity
R = NDArray.eye(3, 5)     # 3x5 matrix with diagonal ones
```

#### NDArray.asarray

Create an array from a nested list or tuple of values.

```python
vec = NDArray.asarray([1, 2, 3, 4, 5])
mat = NDArray.asarray([[1, 2, 3],
                       [4, 5, 6]])
```

#### dtype Parameter

The `dtype` parameter specifies the element type. The default is `Integer`.

```python
int_arr = NDArray.zeros((3, 3))                 # Integer elements
float_arr = NDArray.zeros((3, 3), dtype=Float)  # Float elements
```

### NumPy Interop

You can convert from NumPy arrays for testing in standard Python:

```python
import numpy as np

np_arr = np.array([[1, 2], [3, 4]])
z_arr = NDArray.asarray(np_arr)
```

This is useful for writing tests that run in both Python and the Zinnia compiler.

### Indexing

#### Single-Element Access

```python
val = vec[0]           # first element of a 1-D array
val = mat[1, 2]        # element at row 1, column 2
val = tensor[0, 1, 2]  # element in a 3-D array
```

#### Slice Access

```python
sub = vec[1:4]         # elements 1, 2, 3
sub = vec[::2]         # every other element
sub = vec[::-1]        # reversed

row = mat[0, :]        # first row
col = mat[:, 0]        # first column
block = mat[0:2, 1:3]  # 2x2 sub-matrix
```

#### Stepped Slicing

```python
sub = vec[0:10:2]      # elements at indices 0, 2, 4, 6, 8
sub = mat[::2, ::2]    # every other row and column
```

### Properties

```python
s = arr.shape      # tuple of dimension sizes, e.g. (3, 4)
d = arr.dtype      # element type: Integer or Float
n = arr.ndim       # number of dimensions, e.g. 2
z = arr.size       # total number of elements, e.g. 12
f = arr.flat       # flat (1-D) view of the array
t = arr.T          # transpose of the array
```

## DynamicNDArray -- Bounded-Dynamic Arrays

`DynamicNDArray` is used when the array shape depends on runtime values (such as
private inputs) but compile-time upper bounds exist.

### Compile-Time Envelope

Every `DynamicNDArray` is characterized by two static bounds:

- **L (max elements)**: the maximum total number of elements the array can hold.
- **R (max rank)**: the maximum number of dimensions.

The circuit is generated to accommodate the worst case (L elements, R dimensions). At
runtime, the actual shape may use fewer elements and fewer dimensions.

### Metadata

A `DynamicNDArray` carries runtime metadata that describes its actual shape:

| Field           | Description                              |
|-----------------|------------------------------------------|
| Logical length  | Total number of active elements          |
| Rank            | Actual number of dimensions              |
| Shape vector    | Size of each dimension                   |
| Stride vector   | Step size for each dimension             |
| Offset          | Base index into the underlying storage   |

### Interface

`DynamicNDArray` exposes the same operations as `NDArray`: indexing, slicing,
reductions, reshaping, and element-wise operations. The compiler generates guarded
constraints that account for the dynamic sizing.

```python
def process(n: Public[Integer], data: Private[DynamicNDArray]) -> Integer:
    total = data.sum()
    return total
```

### When to Use DynamicNDArray

Use `DynamicNDArray` when:

- The shape of the array depends on a runtime value (e.g., a private input).
- You can establish an upper bound on the total number of elements.
- You need the flexibility of runtime sizing but still want to generate a valid circuit.

Use `NDArray` when:

- The shape is known at compile time.
- You want the most efficient circuit (no overhead from dynamic sizing).

## Array Operations

Both `NDArray` and `DynamicNDArray` support the following operations.

### Reductions

```python
total = arr.sum()              # sum of all elements
total = arr.sum(axis=0)        # sum along axis 0
product = arr.prod()           # product of all elements
largest = arr.max()            # maximum element
smallest = arr.min()           # minimum element
idx_max = arr.argmax()         # index of maximum element
idx_min = arr.argmin()         # index of minimum element
all_true = arr.all()           # True if all elements are truthy
any_true = arr.any()           # True if any element is truthy
```

### Shape Manipulation

```python
t = arr.transpose()            # transpose (reverses axes)
t = arr.T                      # shorthand for transpose

m = arr.moveaxis(0, 1)         # move axis 0 to position 1
r = arr.reshape((2, 6))        # reshape to new dimensions
f = arr.flatten()              # flatten to 1-D
rep = arr.repeat(3)            # repeat elements
```

### Type Conversion

```python
float_arr = int_arr.astype(Float)      # convert integers to floats
int_arr = float_arr.astype(Integer)    # convert floats to integers
```

### Filtering

```python
positives = arr.filter(lambda x: x > 0)
```

### Conversion

```python
lst = arr.tolist()             # convert to a Python list
```

### Dimensional Properties

```python
n = arr.ndim                   # number of dimensions
s = arr.size                   # total element count
f = arr.flat                   # flat iterator / 1-D view
```

## Broadcasting

Zinnia supports NumPy-style broadcasting for element-wise operations between arrays of
compatible shapes.

### Broadcasting Rules

1. If two arrays have different numbers of dimensions, the shape of the smaller array
   is padded with ones on the left.
2. Arrays with a size of 1 along a particular dimension act as if they had the size of
   the array with the largest shape along that dimension.
3. If two arrays have different sizes along a dimension and neither is 1, a compile-time
   error is raised.

### Examples

```python
# Scalar broadcast: add a scalar to every element
result = arr + 5

# Vector-matrix broadcast: add a row vector to each row
vec = NDArray.asarray([1, 2, 3])           # shape (3,)
mat = NDArray.zeros((4, 3))                # shape (4, 3)
result = mat + vec                         # shape (4, 3)

# Column broadcast
col = NDArray.asarray([[1], [2], [3]])     # shape (3, 1)
mat = NDArray.zeros((3, 4))                # shape (3, 4)
result = mat + col                         # shape (3, 4)
```

## Slicing and Indexing Semantics

### Integer Indexing

Integer indexing reduces the dimensionality by one along the indexed axis.

```python
mat = NDArray.zeros((3, 4))
row = mat[0]          # shape (4,) -- first row
val = mat[0, 0]       # scalar -- single element
```

### Slice Indexing

Slice indexing preserves dimensionality but may change the size along the sliced axis.

```python
mat = NDArray.zeros((5, 5))
sub = mat[1:3]        # shape (2, 5) -- rows 1 and 2
sub = mat[:, 2:4]     # shape (5, 2) -- columns 2 and 3
```

### Mixed Indexing

You can combine integer and slice indices.

```python
mat = NDArray.zeros((4, 5, 6))
sub = mat[0, :, 1:3]  # shape (5, 2) -- fix first and slice third axis
```

### Assignment via Indexing

You can assign to indexed positions:

```python
arr[0] = 99
mat[1, 2] = 42
arr[0:3] = NDArray.asarray([10, 20, 30])
```
