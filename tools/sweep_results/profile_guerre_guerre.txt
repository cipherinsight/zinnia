=== diagnosis ===
guerre triggers ZERO `require_static_int` calls during compile, so the
SMT layer is entirely cold. The on/off delta on a single-process driver
is well within run-to-run noise — the canonical claim of "66× slowdown
on guerre" from `tools/sweep_results/p3_layered_on.json` does NOT
reproduce against the same compiler commit when measured one process at
a time. Most plausible cause for the historical reading: process-pool
contention (sweep_benchmarks.py runs cpu_count // 2 ≈ 6 workers in
parallel; one heavyweight benchmark like perm or guerre running while
others are compiling can stretch wall-clock disproportionately). The
SMT pipeline itself is uninvolved.

The optim-phase resolver is rebuilt by every IRPass via
`builder.export_ir_graph()` (the freshly-built graph has the default
StaticOnlyResolver), so optim-phase queries through this benchmark also
run at zero. AST→IR phase telemetry survives across optim passes via
the `astgen_telemetry` slot wired in lib.rs's `run_pass_pipeline`.
